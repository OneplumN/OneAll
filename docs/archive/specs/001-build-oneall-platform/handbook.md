## 1. 背景与目标

OneAll 是以自研探针体系为核心、支持插件化扩展的智能运维平台，聚焦拨测执行、监控集成、资产聚合与统一配置管理，形成与 ITSM/告警中心联动的运维闭环。

目标包括：即时拨测、持续拨测审批闭环、资产与统计分析、工具与知识沉淀、统一系统设置与审计日志。

## 2. 用户故事与验收

### US1 值班人员即时拨测（P1）
- 目标：快速验证域名/服务可达性与证书有效性。
- 验收要点：
  - HTTP(S) 拨测返回响应时间、状态码、链路日志、节点信息，保留最近一次结果。
  - 证书检测展示到期日、主体信息与链完整性。

### US2 管理员持续拨测与集成（P1）
- 目标：持续拨测申请与 ITSM 审批闭环，外部监控源集成。
- 验收要点：
  - 提交申请后自动创建 ITSM 工单并展示审批状态。
  - 回调通过后自动创建监控任务，集成中心显示插件健康。

### US3 资产与统计分析（P2）
- 目标：统一资产中心与统计报表。
- 验收要点：
  - 资产同步并标记来源与时间。
  - 统计报表包含成功率、响应时间、节点分布，支持导出。

### US4 工具与知识沉淀（P3）
- 目标：工具库、脚本复用、知识库管理。
- 验收要点：
  - 工具执行记录可追溯并可回写知识库。
  - 知识库按标签/关键词检索并展示最近编辑信息。

## 3. 功能需求（FR）

1. FR-001：基于角色的访问控制，细分监控、资产、工具、知识等模块权限。  
2. FR-002：首页概览蜂窝图、探针健康、告警摘要、待办事项，刷新延迟 ≤ 60 秒。  
3. FR-003：支持 HTTP(S)/Telnet/WSS 等协议的即时拨测，多节点并发。  
4. FR-004：一次性拨测包含响应时间、状态码、链路日志、节点信息，保留最近结果。  
5. FR-005：证书检测返回到期时间、主体信息、链完整性与风险提示。  
6. FR-006：提供 CMDB 域名校验与修复提示。  
7. FR-007：集成中心支持拨测工具插件化扩展。  
8. FR-008：持续拨测申请表单字段完备且强校验。  
9. FR-009：自动创建 ITSM 工单并展示审批时间轴。  
10. FR-010：审批通过后创建拨测任务，配置频率/阈值/通知。  
11. FR-011：拨测任务展示执行记录、告警次数、关联工单与操作日志。  
12. FR-012：统计分析提供成功率、平均响应时间、节点分布、趋势对比与导出。  
13. FR-013：插件化集成外部监控源并展示健康状态。  
14. FR-014：资产中心聚合域名/监控主机/项目/工单资产并支持导出。  
15. FR-015：资产同步支持手动/定时、同步日志与来源标记。  
16. FR-016：工具库支持脚本绑定、标签、执行历史，联动探针调度。  
17. FR-017：代码管理支持脚本版本历史、在线编辑、标签分类。  
18. FR-018：集成中心支持外部接口配置、插件启停、健康检查与告警。  
19. FR-019：探针管理展示心跳状态、节点属性、告警与日志。  
20. FR-020：知识库支持权限控制、分类标签、全文检索与图文/代码展示。  
21. FR-021：系统设置涵盖用户、权限、日志、通知、主题与外部接口配置。  
22. FR-022：统一操作与访问日志，可检索与导出。  
23. FR-023：全模块结构化日志与审计记录。

## 4. 关键实体与数据模型

核心实体（节选）：
- User：统一身份、角色与审计关联。
- ProbeNode：探针节点，记录位置、网络属性、协议与心跳。
- DetectionTask：一次性拨测任务与结果。
- MonitoringRequest / MonitoringJob：持续拨测申请与周期任务。
- MonitoringSource：外部监控源配置与健康状态。
- AssetRecord：资产中心数据记录与来源追踪。
- ToolDefinition / ScriptVersion：工具与脚本版本管理。
- KnowledgeArticle：知识库文章。
- AuditLog：操作与访问审计记录。

关键业务规则（摘要）：
- 探针离线时任务暂停并通知责任人。
- ITSM 回调失败自动重试，仍失败进入补偿队列。
- 资产同步支持全量/增量并保留历史版本。
- 重要日志需审计、不可变与可导出。

数据保留策略（摘要）：
- 拨测原始日志归档，指标保留一年级别。
- 审计与审批记录保留 3 年。
- 知识库保留最近 5 个版本。

## 5. 技术与架构决策

- 前端：Vue 3 + TypeScript + Vite + Element Plus。
- 后端：Python 3.11 + Django 4.2 + DRF；Celery 5.x + Redis。
- 存储：MySQL 8（事务）+ TimescaleDB 2.x（时序）+ Redis（缓存/队列）。
- 可观测：结构化日志 + Prometheus 指标；链路追踪暂不启用。
- 探针通信：HTTP 起步，预留 gRPC 扩展。
- 部署：Docker Compose；Ansible 为可选自动化；不引入 K8s。

## 8. 快速开始（摘要）

详细步骤与完整命令见 `quickstart.md`。以下为要点：

1) 准备依赖：Node.js 20 + pnpm、Python 3.11 + pipx、Docker Compose；可选 Ansible。  
2) 克隆并配置环境变量：`cp backend/.env.example backend/.env`。  
3) 启动基础服务：MySQL、TimescaleDB、Redis。  
4) 启动后端 + Celery、前端开发服务。  
5) 启动探针并验证拨测流程。  

注：探针主用实现以 `probes/README.md` 为准，快速开始中的历史命令若不匹配请回溯更新。
