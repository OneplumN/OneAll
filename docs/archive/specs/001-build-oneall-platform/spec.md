## User Scenarios & Testing *(mandatory)*

### User Story 1 - 运维值班人员即时拨测 (Priority: P1)

运维值班人员需要快速验证线上域名或服务可达性，以便在收到告警时立即定位问题。

**Why this priority**: 直接决定故障响应效率，是平台的首要价值。

**Independent Test**: 在沙箱环境中使用任意域名发起一次性拨测，验证多协议支持与返回数据完整性。

**Acceptance Scenarios**:

1. **Given** 值班人员已登录且拥有拨测权限, **When** 选择探针节点对目标域名发起 HTTP(S) 拨测, **Then** 平台返回响应时间、状态码、链路日志并保留最近一次结果。
2. **Given** 值班人员需要验证证书有效性, **When** 在拨测工具中选择证书检测, **Then** 平台展示证书到期日、主体信息与链完整性。

---

### User Story 2 - 平台管理员建立持续拨测与集成 (Priority: P1)

平台管理员希望为关键业务域名建立长期拨测，联动 ITSM 审批并管理插件化监控源。

**Why this priority**: 持续拨测和集成能力是平台与现有系统协同的核心。

**Independent Test**: 提交一份拨测监控申请并启用外部监控插件，验证审批流程、任务创建与插件健康展示。

**Acceptance Scenarios**:

1. **Given** 管理员在申请表单填写域名、阈值等必填项, **When** 提交申请, **Then** 平台校验字段并自动创建 ITSM 工单。
2. **Given** ITSM 回调审批通过, **When** 平台接收回调, **Then** 自动生成拨测任务并在集成中心显示关联插件状态。

---

### User Story 3 - 数据分析与资产管理人员聚合信息 (Priority: P2)

分析人员需要在统一界面掌握拨测统计、资产信息及工具执行效果，以支撑趋势分析与合规汇报。

**Why this priority**: 数据聚合能力直接影响跨团队协同与决策。

**Independent Test**: 同步资产数据源并导出拨测报表，验证资产覆盖率与统计视图完整性。

**Acceptance Scenarios**:

1. **Given** 平台配置了域名、主机与项目等资产来源, **When** 人员手动触发同步, **Then** 资产中心展示最新字段并标记数据来源与时间。
2. **Given** 拨测任务已运行产生数据, **When** 人员选择时间范围导出统计报表, **Then** 平台生成包含成功率、响应时间、节点分布的文件。

---

### User Story 4 - 运维团队共享工具、脚本与知识 (Priority: P3)

运维工程师希望在平台内集中管理脚本、工具和经验内容，实现快速复用与响应。

**Why this priority**: 工具与知识沉淀提升处理效率，减少重复劳动。

**Independent Test**: 创建一个脚本并关联工具库，发布知识库文章并验证权限与检索。

**Acceptance Scenarios**:

1. **Given** 工程师上传脚本并设置分类标签, **When** 在工具库触发执行, **Then** 平台记录执行日志并允许结果回写知识库。
2. **Given** 知识库存在多篇文章, **When** 用户按标签或关键词检索, **Then** 平台返回匹配内容并展示最近编辑信息。

### Edge Cases

- ITSM 回调超时或失败时，申请状态必须退回并提示人工跟进，同时保留重试入口。
- 探针节点离线或负载饱和时，拨测操作需被阻止并提示可用节点或排队策略。
- 外部资产源返回重复、缺失或冲突字段时，平台需保留历史数据、标记异常并允许人工确认。
- 插件健康检查失败时，平台需自动告警并在集成中心显示故障原因与处理步骤。
- 知识库或工具执行涉及敏感信息时，必须记录访问与操作审计。
- 首页告警摘要或待办数据获取失败时需提供降级提示与刷新入口，若延迟超过 2 分钟则触发运维通知。

### Terminology

- **集成中心**：集中管理插件配置、外部监控源状态及健康指标的统一入口（对应 FR-007、FR-018 及 Phase 5 相关任务）。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 平台必须提供基于角色的访问控制，细分监控、资产、工具、知识等模块权限。
- **FR-002**: 平台必须在首页提供可视化概览，展示拨测任务蜂窝图、探针健康、告警摘要（含告警等级、来源与最近触发时间）和待办事项（覆盖 ITSM 待审批、探针异常排查等入口），并保证数据刷新延迟不超过 60 秒。
- **FR-003**: 系统必须支持域名即时拨测，覆盖 HTTP(S)、Telnet、WebSocket 等协议，并允许多节点并发测试。
- **FR-004**: 一次性拨测结果必须包含响应时间、状态码、链路日志、节点信息，并保留最近一次结果供复查。
- **FR-005**: 系统必须支持证书检测，返回到期时间、主体信息、链完整性及风险提示。
- **FR-006**: 系统必须提供 CMDB 域名校验，标记存在或缺失并提示负责人与修复建议。
- **FR-007**: 平台必须允许通过“集成中心”扩展拨测工具，新增工具需配置入口、参数和结果展示模板。
- **FR-008**: 拨测监控申请表单必须涵盖域名、所属系统、网络类型、负责人、告警阈值、节点选择、通知策略，并强制校验。
- **FR-009**: 平台必须自动创建 ITSM 工单并展示审批状态时间轴，支持审批通过、退回、驳回等回调。
- **FR-010**: 审批通过后系统必须创建拨测任务，配置执行频率、阈值、通知渠道，并提供任务生命周期视图。
- **FR-011**: 平台必须展示拨测任务执行记录，包括最近成功/失败、告警次数、关联工单与操作日志。
- **FR-012**: 统计分析模块必须提供拨测成功率、平均响应时间、节点分布、趋势对比，并支持图表与数据导出。
- **FR-013**: 平台必须允许通过插件集成多监控源（如外部监控平台），在监控模块展示其健康状态与主机组概览。
- **FR-014**: 资产信息中心必须聚合域名资产、外部监控主机、项目资产、工单纳管主机，并支持搜索、筛选、导出。
- **FR-015**: 资产信息同步必须支持手动触发、定时同步、查看同步日志，并保留数据来源与更新时间。
- **FR-016**: 工具库必须支持脚本与工具绑定、分类、标签、执行历史记录，并允许与探针调度联动。
- **FR-017**: 代码管理模块必须提供脚本版本历史、在线编辑、标签分类，并供其他模块调用执行。
- **FR-018**: 集成中心必须支持管理外部系统接口配置、插件启停、健康检查与告警通知参数。
- **FR-019**: 探针管理模块必须展示探针心跳状态、节点属性，并支持告警与日志查看。
- **FR-020**: 知识库必须提供多用户维护、权限控制、分类标签、全文检索、图文内容与代码片段展示。
- **FR-021**: 系统设置必须涵盖用户管理、权限管理、系统日志、通知配置、插件控制、外部接口配置、平台主题等项。
- **FR-022**: 平台必须提供统一的操作与访问日志，可按用户、时间、模块检索并导出。
- **FR-023**: 所有模块必须支持结构化日志与审计记录，以便追踪执行与问题定位。

### Key Entities *(include if feature involves data)*

- **ProbeNode**: 可调度探针节点，记录地理位置、网络属性、支持协议、心跳状态。
- **DetectionTask**: 一次性拨测或证书检测任务，记录目标、协议、节点、结果、执行时间。
- **MonitoringRequest**: 持续拨测申请，包含表单字段、审批记录、关联 ITSM 工单号。
- **MonitoringJob**: 审批通过后生成的周期拨测计划，记录频率、阈值、节点、最近执行状态。
- **MonitoringSource**: 外部监控插件或集成源，记录类型、启用状态、健康检查结果。
- **AssetRecord**: 资产数据条目，包含域名/主机/项目属性、来源标签、同步时间、责任人。
- **ToolDefinition**: 工具库中配置的工具，关联脚本、参数模板、分类标签与执行历史。
- **ScriptVersion**: 代码管理中的脚本版本，记录内容摘要、作者、更新时间、关联工具。
- **KnowledgeArticle**: 知识库文章，包含分类、标签、权限范围、内容与最近编辑信息。
- **AuditLog**: 操作与访问日志，记录用户、动作、目标模块、结果与时间戳。

## Assumptions & Dependencies

- ITSM 平台提供标准接口用于创建工单并在审批后回调结果。
- 外部监控、资产系统与知识库数据源提供可调用的同步接口或文件导出能力。
- 探针节点由运维团队部署并提供健康检查接口。
- 公司统一身份与权限平台可供接入，减少账户重复维护。
- 插件生态初期以内部维护为主，后续可开放第三方接入机制。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 80% 的一次性拨测在 3 分钟内完成并生成可供排障的日志信息。
- **SC-002**: 关键业务域名的拨测监控申请在 1 个工作日内完成审批或给出处理意见。
- **SC-003**: 资产信息中心覆盖率达到 95%，并能在 24 小时内同步外部数据变更。
- **SC-004**: 平台上线后三个月内运维满意度调查得分达到 4/5 以上。
- **SC-005**: 统计分析模块能在 2 分钟内生成最近 30 天的报表，并支持 CSV/Excel 导出。
- **SC-006**: 工具库与代码管理模块的复用率在上线三个月后达到 70%（至少 7 成工单可通过平台工具处理）。
- **SC-007**: 平台工作日需保持业务可用，计划维护窗口限定在周六至周日执行。
