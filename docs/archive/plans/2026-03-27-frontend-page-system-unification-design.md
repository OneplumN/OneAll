# Frontend Page System Unification Design

**Date:** 2026-03-27

## Context

当前前端已经有部分共享样式能力，但页面层没有形成真正的设计系统约束。

已确认的核心问题：

- 普通业务页存在多套平行页面壳：
  - `frontend/src/components/RepositoryPageShell.vue`
  - `frontend/src/pages/settings/components/SettingsPageShell.vue`
  - `frontend/src/pages/settings/components/SettingsRepositoryShell.vue`
- 同样的业务动作在不同模块里表现不一致：
  - 刷新有的做成卡片，有的做成按钮
  - 主按钮、次按钮高度和留白不一致
  - 筛选栏位置、宽度、排列方式不一致
  - 列表页表格容器和底部分页结构不一致
- 资产中心、监控与告警、系统设置三组页面正在演化成三种不同的页面语言

这意味着当前项目不是“某几个按钮不好看”，而是页面框架、工具栏、表格和底部区没有统一标准。后续继续开发模块，只会不断复制这种不一致。

---

## Scope

本轮统一覆盖所有普通业务页面，包括但不限于：

- 资产中心
- 监控与告警中的普通列表/详情/表单页
- 系统设置
- 工具页
- 探针节点/日志等普通业务页

明确排除：

- `拨测可视化` 页面

原因：

- 该页面已在另一窗口独立推进
- 其信息展示形态属于可视化页，不属于本轮要统一的“普通业务页框架”

---

## Design Goal

建立一套统一、克制、可维护的后台页面系统，使所有普通业务页都遵循同一套结构语言，而不是各写各的。

目标不是重新发明一套全新审美，而是：

- 以资产中心当前相对成熟的结构为骨架
- 把按钮、筛选栏、表格、底部区和页面壳抽象成统一规范
- 让设置页和告警页也说同一种界面语言

---

## Design Direction

本轮采用方案 B：

**重新定义一套统一页面规范，但骨架沿用资产中心这套思路。**

不采用的方案：

- 不直接把监控策略页当全站标准，因为它本质上仍是单页手工拼装
- 不直接把系统设置页当全站标准，因为它保留了独立壳体系
- 不只改主题 token，因为当前问题已经不仅是颜色和字号，而是结构层问题

---

## Page Frame Standard

所有普通业务页统一使用同一种页面结构：

1. 顶部标题栏
2. 工具栏
3. 主内容区
4. 底部区

### 1. 顶部标题栏

- 左侧只承载：
  - 模块名
  - 当前页名
  - 可选 breadcrumb
- 右侧只承载：
  - 刷新
  - 主操作
  - 少量全局动作
- 标题栏高度、左右内边距、底部分隔线全站一致
- 页面不能自行定义另一套 header 排版

### 2. 工具栏

- 标题栏下方单独一行
- 左侧放业务主动作：
  - 新建
  - 导入
  - 批量操作
  - 导出
- 右侧放筛选与搜索
- 所有筛选项平行排列，不使用上下两层布局
- 搜索框固定在右侧最后一个

### 3. 主内容区

- 列表页默认就是一个主表格容器
- 不鼓励页面再套多层卡片
- 滚动行为统一，避免“页面滚”和“表格滚”混杂

### 4. 底部区

- 列表页：左统计，右分页
- 表单页：固定保存/取消区
- 底部边线、高度、内边距统一

一句话总结：

普通业务页统一成长成：

`标题栏 -> 工具栏 -> 主内容 -> 底部区`

## Control Standard

### Buttons

- 主按钮：
  - 仅用于“新建 / 保存 / 确认同步 / 立即执行”等主动作
- 次按钮：
  - 用于“导入 / 导出 / 取消 / 返回 / 历史”
- 危险按钮：
  - 仅用于“删除 / 清空 / 重置”等不可逆动作
- 文字按钮：
  - 仅出现在表格操作列

规则：

- 同一行最多一个主按钮
- 全站按钮高度统一
- 不允许页面自行定义另一套按钮高度和圆角

### Refresh

- 刷新属于页面级动作，不属于业务主按钮
- 固定放在标题栏右侧动作区
- 全站统一成一种视觉样式，不再允许页面自行发明“卡片式刷新”或其他变体

### Filters

- 全站筛选项统一为横向平铺
- 统一两档宽度：
  - 窄筛选：状态、类型、来源
  - 标准筛选：名称、关键词、搜索
- 筛选控件高度一致
- placeholder 风格统一为业务语义，不使用技术性描述

### Search

- 固定在工具栏右侧最后一个
- 宽度统一
- 与筛选框保持同一高度和边框语言

### Action Column

- 统一使用文字动作，不使用小块按钮
- 排序规则：
  - 编辑类在左
  - 查看类在中
  - 危险类在右
- 颜色规则：
  - 编辑：绿色系
  - 详情：主色系
  - 警示/冲突：橙色系
  - 删除：红色系

---

## Table And Footer Standard

### Table

- 全站普通列表表格统一使用 `.oa-table`
- 表头高度、正文高度、字号、padding 固定
- 表格正文不再按“主信息 / 辅助信息”使用不同字号
- 空状态统一
- 表格外层容器统一

### Footer

- 列表页底部固定为：
  - 左：总数 / 统计
  - 右：分页
- 表单页底部固定为：
  - 保存 / 取消
- 统一底部边线与内边距，避免出现双线、贴边、悬空等不一致情况

---

## Source Of Truth

本轮不再允许页面层长期维护多套平行页面壳。

统一后的来源：

- 页面骨架：基于 `frontend/src/components/RepositoryPageShell.vue` 演进出的统一页面壳
- 全局样式 token：`frontend/src/styles/theme.scss`
- 页面级业务内容：各模块页面自身

需要被收敛的重复来源：

- `frontend/src/pages/settings/components/SettingsPageShell.vue`
- `frontend/src/pages/settings/components/SettingsRepositoryShell.vue`
- 告警域页面内部自带的 header / toolbar 局部结构

## Rollout Strategy

本轮按“先底座，后页面”的顺序推进。

### Phase 1: 统一底座

- 补齐统一页面壳能力
- 统一按钮、刷新、筛选、搜索、底部区样式
- 明确共享 class 和 token

### Phase 2: 迁移代表页面

优先迁移：

- 资产中心
- 监控策略
- 告警事件
- 系统设置

这四组页面覆盖了列表页、详情页、表单页和设置型页面，足以成为后续复制模板。

### Phase 3: 扩展到全站普通页

- 设置模块剩余页面
- 工具页
- 探针节点/日志页
- 其他普通列表页

## Risk Assessment

### Low Risk

- 统一按钮高度、颜色、圆角
- 统一筛选栏与搜索框布局
- 统一表格容器与分页区

### Medium Risk

- 壳组件收敛时，设置页滚动行为可能需要单独适配
- 个别页面的 footer 固定行为可能与现有表单滚动方式冲突
- 局部 scoped style 中的硬编码 spacing 可能与新壳冲突

### Not Included In This Round

- 可视化页设计重做
- 品牌色重做
- 图标体系重做
- 深色模式优化
- 页面内容信息架构重构

---

## Acceptance Criteria

- 普通业务页只保留一套页面框架语言
- 标题栏、工具栏、表格容器、底部区布局一致
- 主按钮、次按钮、刷新、筛选、搜索全站表现一致
- 操作列颜色、排序、字体和间距一致
- 系统设置不再保留一套独立的页面壳语言
- 拨测可视化页不受本轮改动影响
