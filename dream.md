# 智能运维平台（V1.0）产品需求文档

## 一、产品概述

智能运维平台旨在构建一个以 **自研探针体系** 为核心、支持 **插件化扩展** 的智能监控与拨测管理平台。
 平台聚焦于 **可观测性、拨测执行、资产数据聚合、插件生态与统一配置管理**，并与公司现有的告警中心和 ITSM 流程联动，形成闭环。

**主要目标：**

- 提供多维度拨测与监控可视化能力
- 构建统一的探针调度与节点执行体系
- 支持多种监控源（Zabbix、Prometheus）的插件扩展
- 统一资产信息管理与同步
- 支撑自定义工具与脚本的可视化调用
- 集成外部系统（ITSM、监控、CMDB等）
- 提供集中配置与权限管理中心

------

## 二、核心功能模块

### 1. 监控模块

**目标：** 统一展示和扩展多监控体系（内置与外部）。

**功能点：**

- 概览（内置）：蜂窝图展示拨测任务与监控状态。
- 支持通过插件集成 Zabbix、Prometheus。
- 展示监控源健康状态、主机组概览。
- 可筛选、聚合和跳转至资产详情。

------

### 2. 一次性拨测检测工具（核心）

**目标：** 提供即时检测能力。

**功能点：**

- **域名拨测**
  - 节点选择（探针同步）
  - 多协议支持（HTTP(S)、Telnet、WSS 等）
  - 保留最近一次检测结果
- **证书检测**
  - 支持探针节点执行
  - 返回证书到期信息、CN、链验证结果
- **CMDB 域名检验**
  - 检查域名是否存在于本地资产模型中
- **扩展性：**
  - 支持通过“集成中心”扩展其他检测工具。

------

### 3. 拨测监控申请

**目标：** 建立持续拨测任务的申请与审批联动机制。

**功能点：**

- 申请表单字段：
  - 域名、所属系统、互联网类型（内/外网）、负责人、告警联系人、节点选择、告警阈值。
- 自动调用 ITSM 接口创建工单。
- 等待 ITSM 回调（审批成功/失败）。
- 任务清单页面展示任务状态与执行详情。

------

### 4. 统计分析

**目标：** 提供基础报表分析与趋势展示。

**功能点：**

- 拨测报表（内置）
- 支持多维度统计（成功率、平均响应时间、节点分布等）
- 可导出报表（CSV / Excel）
- 可通过插件扩展更多统计项。

------

### 5. 资产信息中心

**目标：** 统一展示和同步多来源资产信息。

**功能点：**

- **域名资产（内置）**
  - 字段：域名、所属系统、互联网类型、负责人、告警联系人。
- **Zabbix 主机（扩展）**
  - 字段：IP、主机名称、主机群组、Proxy。
- **IPMP 项目（扩展）**
  - 字段：应用编号、中文名、简称、状态、负责人、等保级别、系统归属。
- **工单纳管主机信息（扩展）**
  - 字段：IP、机房、ITSI Proxy、端口、联系人、hostname、系统信息。
  - 支持 Proxy 名称与 IP 映射配置（通过系统设置）。

**补充功能：**

- 支持单独手动同步外部数据。
- 可查看同步日志与数据来源。
- 支持搜索、筛选与导出。

------

### 6. 工具库

**目标：** 提供可复用的运维工具集合。

**功能点：**

- 工具与脚本绑定，可调用代码管理中的脚本。
- 可用于正则验证、格式转换、快速诊断等操作。
- 支持分类与标签管理。

------

### 7. 代码管理

**目标：** 平台脚本统一管理与共享。

**功能点：**

- 版本控制与历史记录。
- 在线文本编辑器（支持多语言）。
- 标签与分类管理。
- 其他模块可直接调用执行。

------

### 8. 集成中心（核心）

**目标：** 插件化扩展与统一管理。

**功能点：**

- 插件类型：
  - 监控插件（概览 / Zabbix / Prometheus）
  - 检测插件（域名拨测 / 证书检测 / CMDB 检验）
  - 报表插件（拨测统计）
  - 资产插件（域名 / Zabbix 主机 / IPMP / 工单主机）
  - 工具插件
  - 模板中心
- 插件状态控制（启用 / 禁用 / 删除）
- 内置插件不可删除，扩展插件可管理。
- 插件可配置导航栏显示。

------

### 9. 探针系统（核心）

**目标：** 管理自研探针节点与任务执行。

**功能点：**

- 节点管理：探针ID、IP、名称、机房、心跳状态、运行时长。
- 调度管理：任务下发策略（随机、负载、指定节点）。
- 策略配置：拨测频率、超时时间。
- 日志查看：探针执行与结果日志。
- 与拨测监控申请、一次性拨测联动执行。

------

### 10. 知识库

**目标：** 运维经验沉淀与检索中心。

**功能点：**

- 多人维护、权限控制。
- 搜索与分类标签。
- 支持文本、图片、代码段。
- 底层存储使用关系型数据库（MySQL），支持按标题/正文检索。

------

### 11. 系统设置（强化版）

**目标：** 平台综合配置与系统管理中枢。

**功能点：**

```
系统设置
 ├── 用户管理（支持 LDAP 同步）
 ├── 权限管理（基于角色/模块）
 ├── 系统日志（操作/访问）
 ├── 告警通知配置（第三方推送参数）
 ├── 探针与 Proxy 配置（节点映射、心跳周期）
 ├── 插件管理（开关控制）
 ├── 外部系统接口配置（ITSM、Zabbix、Prometheus）
 ├── 平台主题与基础信息（Logo、主题、系统名）
```

------

## 三、用户界面布局

### 1. 全局布局

- 顶部导航栏：系统名称 + Logo（左），用户信息（右）。
- 左侧导航栏：显示当前模块的二级菜单，可折叠。
- 右侧主内容区：模块主体内容展示。

### 2. 一级导航结构

```
监控
一次性拨测检测工具
拨测监控申请
统计分析
资产信息
工具库
代码管理
集成
探针
知识库
系统设置
```

### 3. 交互说明

- 模块间通过统一样式与权限体系切换。
- 支持多标签页浏览（可选）。
- 所有表格支持筛选、导出、分页。
- 插件模块可动态加载界面组件。

------

## 四、技术架构与实现考量

### 前端

- **框架**：Vue 3 + Element Plus
- **特性**：
  - 动态路由加载（支持插件扩展）
  - 统一主题配置
  - 模块化组件封装（监控图表、表格组件）

### 后端

- **框架**：Python + Django
- **服务结构**：
  - 插件注册机制（插件配置表 + 路由动态注册）
  - 任务调度：Celery + Redis
  - 探针通信：HTTP 或 gRPC
  - 数据模型：ORM 统一封装

### 中间件

| 功能       | 推荐组件           |
| ---------- | ------------------ |
| 时序数据   | TimescaleDB        |
| 缓存与队列 | Redis              |
| 关系数据   | PostgreSQL / MySQL |

### 部署与扩展

- 模块化部署（前后端分离）
- 探针支持独立部署与分布式扩展
- 插件通过配置文件注册，可热更新
- 与外部系统交互通过 REST API

------

## 六、总结

**平台定位：**

> 一个以探针体系为核心、插件化扩展为特色的智能运维基础平台，
>  聚合拨测、监控、资产、知识与工具，
>  形成“检测—展示—配置—扩展”四层结构，
>  并与公司现有的告警中心、ITSM体系形成无缝集成。

该版本聚焦于基础可观测性与平台架构打底，为后续智能化与自动化迭代（V2.0）打下坚实基础。
